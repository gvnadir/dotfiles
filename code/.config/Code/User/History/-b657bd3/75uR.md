## Controlling File and Directory Permissions

### Granting Permissions

There are three levels of permissions:

- **r:** Permission to read. This grants permission only to open and view a file
- **w:** Permission to write. This allows users to view and edit a file
- **x:** Permission to execute. This allows users to execute a file (but not necessarily view or edit it)

These are valid of directories:

- **r:** Permission to read. This allows a user to list the contents of the directory and subdirectories using commands like `ls`
- **w:** Permission to write. This allows a user to create, delete, or rename files diectories wiithin the directory.

- **x:** This is required to access the contents of a directory using commands like `cd``.

### Granting Ownership to an Individual User

To move ownership of a file to a different user we can use the `chown`(change owner) command:

`chown user file`

### Granting Ownership to a Group

To transfer ownership of a file from one group to another, we can use the `chgrp` (change group) command:

`chgrp group file`

### Changing Permissions

#### Changing Permission with Decimal Notation

To change the permission of a file we can use `chmod` (Change Mode).

```
# this command will change the permission of the file to
# rwxr--r--
# 111 = 7
# 100 = 4
# 100 = 4
chmod 744 hashcat.hcstat
```

#### Changing Permissions with UGO

UGO stands for User Group Others. This is another commond way for changing permissions using the following operatoros:

- \- Removes a permission
- \+ Adds a permission
- = Sets a permission

```
# add writing permissions for user to the file
chmod u+w hashcat.hcstat

# add execute permissions for both user and other
chmod u+x, o+x hashcat.hcstat
```

### Default Permissions

Linux automatically assigns all file and directories default permissions of 666 and 777 respectively. This means that, by default, you won't be able to execute a file immediately after downloading it. If you try you'll usually get a message that says something like "Permission denied".

For these cases, you'll need to give yourself root and execute permissions using `chmod`.
The default permissions allocated to files and directories can be changed using the `umask` (user file-creation mask) command.

The `umask` is a three-digit octal number corresponding to the three permissions digit, but the `umask` number is _subtracted_ to the permissions number.

For example, if the `umask` is set to 022, a nr file with original default permissions of 666 will now have the permissions 644. In Kali, as with most Debian systems, the `umask` is preconfigured to 022, meaning the Kali default is 664 for files and 755 for directories.

### Granting Temporary Root Permissions with SUID

The SUID special permission stands for Set User ID. Basically, the SUID bit says that any user can **_execute_** the file with the permissions of the owner but those permissions don't extend beyond the use of that file.

So, basically, if you have a program (an executable file) that contains some line of code that requires root permissions but we want everyone (other) to execute it without having the root priviledges, Root can set the SUID to that file. This happens for different binaries like the `/bin/passwd`, because everybody needs to change its own password, even without root priviledges (required in the line of code though).

To set the SUID, just add 4 using the `chmod` with the decimal notation: `chmod 4744 file`.

Let's do a practical example:

1. Create a new linux user: `newuser`.
2. Create its home directory `/home/newuser` and assign it to him.
3. Set a password for that user.

```
sudo useradd newuser
sudo mkdir /home/newuser
sudo chown newuser:newuser /home/newuser
sudo passwd newuser
```

Since I didn't add `newuser` to the sudo group, it doesn't have root priviledges at all. To add it to the sudo group just run `sudo usermod -aG sudo user`, but for the sake of the example we won't.

Now let's create a C program that prints out the User ID (the user identity) and the Effective User ID (the user's priviledges assigned for the process). Furthemore, the code attemps to create a directory in root `/` that requires root priviledges to be done.

> **The SUID won't work with scripts (bash/python/etc) but only with compiled code like C etc.**

```
#include <stdio.h>
#include <syslog.h>
#include <unistd.h>
#include <sys/stat.h>

int main() {
    printf("User ID: %d\n", getuid());
    printf("Effective User ID: %d\n", geteuid());

    // Attempt to create a directory in the root directory
    if (mkdir("/mydir", S_IRWXU | S_IRWXG | S_IRWXO) == 0) {
        printf("Directory created successfully.\n");
    } else {
        perror("Failed to create directory");
    }
    return 0;
}

```

Now, as root, let's compile the `program.c` file using `gcc program.c -o program` and let's set the SUID to the compiled file using `chmod 4755 program`.

> **the `other` user must have the `x` permission on the file in order to execute it.**

A `s` in the `user`'s permission part will indicate that the bit was successfully set:

`-rwsr-xr-x 1 root root 16208 Oct  8 16:34 program`

now let's move the file to `/home/newuser` (so he can run it), switch to the `newuser` user and execute the code:

```
mv program /home/newuser
su - newuser
./program

#output
User ID: 1001
Effective User ID: 0
Directory created successfully.
```

The output shows that the User ID and the Effective User ID are different, pointing out that we executed the file as `newuser` but we were root user the moment the process was running! In fact, we successfully created the `mydir` folder into root (`/`) without having root priviledges set to `newuser`! (we temporarily borrowed them just for executing the code).

### Granting the Root User's Group Permissions SGID

SGID also grants temporary elevated permissions, but it grants the permissions of the file owner's group, rather than of the file's owner.

It's possibile to set the SGID bit using the number 2 with `chdmod`.

> **When it comes to directories, the SGID works differently. The ownership of new files created in that directories goes to the directory creator's group, rather than the file creator's group**

### The Sticky Bit

The _sticky bit_ is a permission bit that you can set on a directory to allow a user to delete or rename files within that directory.

### Example for finding files based on permissions

`find / -user root -perm -4000`

this command will find all the root user's files with the SUID bit set on those files. Zeros are considered placeholders in the `find` command.

### Changing Process Priority with nice and renice

The nice priority values start from -20 to 19 with 0 as default value.

The nicer you are with _other_ users the less the resources you want to use. It means that that higher is the nice value for a process, the less the priority assigned to it.

```
# increment the value of slowprocess by 10
# you're being nice
nice -n 10 /bin/slowprocess

# increment the value of slowprocess by -10
# you're not being nice
nice -n -10 /bin/slowprocess
```

The renice command takes absolute values between -20 and 19 and sets the priority to that particular level, rather than increasing or decreasing from the level at which it started.

In addition, renice required the PID od the process rather than its name.

```
# set the nice value of the process 6996 to 19
renice 19 6996
```

## Compressing and Archiving

Compression can be generally categorized as either _lossy_ or _lossless_.  
Lossy compression is very effective in reducing the size of files, but the integrity of the information is lost, in other words, the file compression is not exactly the same as the original.

Lossy compression works great for graphics, video and audio files, where a small difference is hardly noticeable.

If your are sending script or document, the integrity of the original file myst be retained when it is decompressed, that's why we will use lossless compression.

### Tarring Files Together

_Tar_ stands for _tape archive_, a reference to the prehistoric days of computing where systems used tape to store data. The `tar` command creates a single file from many files, which is then referred to as an _archive_, _tar file_ or _tarball_.

```
# creating a tarball of three files

ls
hackersarise1.sh hackersarise2.sh hackersarise3.sh

# -c create, -v verbose, -f write to the following file
tar -cvf HackersArise.tar hackersarise1 hackersarise1 hackersarise2


# -t list content
tar -tvf HackersArise.tar

# -x extract

tar -xvf HackersArise.tar
```

### Compressing Files

The common Linux commands for compressing files:

- `bzip2`, which uses the extension _.tar.bz2_, is the slowest, but the resultant files are the samllest
- `compress`, which uses the extension _.tar.z_, is the fastest, but the resultant files are larger
- `gzip`, which uses the extension _.tar.gz_ or _tgz_, falls somewhere in between the two commands above

```
# compress file using gzip

# uncompress file using gzip

gunzip HackersArise.*

# compress file using bzip2

# uncompress file using bzip2

## Filesystem and Storage Device Management

### The Device Directory /dev

Linux has a special directory that contains files representing each attached device: the /dev directory.

The devices _sda1_, _sda2_, _sdb_, _sdb1_, are usually the hard drive or USB flash drive and its partitions.

### How Linux Represents Storage Devices

Originally, Linux represented floppy drives as _fd0_ and hard drives as _hda_. Today, hard drives are repsented as _sda_ that stands for SATA (Serial Advanced Techology Attachment), the newer and faster version of the old ATA. SATA and ATA are both interface standards used to communicate with and manage data transfer to and from hard drives and other storage devices.

> An interface is a connection or interaction point between different components or systems, allowing them to communicate and exchange information. It defines how they can work together, making it possible for one part to understand and interact with another.

When a system have more than one hard drive, Linux simply names them serially by incrementing the last letter in alphabetical order, so the first drive is _sda_, and the second drive is _sdb_ and so on.
Since some drives can be split into partitions in order to manage and separate information, the first partition on the first SATA drive would be _sda1_, the secondo partition would be _sda2_ and so on.

### File Systems

A file system is a software component or protocol that controls how data is stored, accessed, and managed on a storage device. It establishes the structure for naming files and organizing them into directories, as well as managing metadata associated with each file.

Different operating systems use various file systems, such as:

- FAT (File Allocation Table), NTFS (New Technology File System) in Windows
- ext4 (Extended File System version 4) in Linux
- HFS, HFS+ (Hierarchical File System) in macOS.

### Character and Block Devices

Something else to note about the naming of device files in the /dev directory is that the first position contains either _c_ or _b_.

These letters represent the two ways that devices transfer data in and out. The _c_ stands for character, and these devices are known as _character_ devices such as keyboards.

The _b_ stands for _block_ devices, the communicate in blocks of data (multiple bytes at a time) and include devices like hard drives and DVD drives. These devices require higher-speed data throughput and therefore send and receive data in blocks.

```
# to see block devices use the 'lsblk' command (list block)

lsblk

# to get info the filesystem type use the '-fs' switch

lsblk -fs
```

### Mounting and Unmounting

A storage device must be first _physically_ connected to the filesystem and then _logically_ attached to the filesystem in order for the data to be made available in the operating system.

The command _mount_ is used to _logically_ attach a device to the OS. The two main mount points in Linux are _/mnt_ and _/media_.

```
# mount a device

mount /dev/sdb1 /mnt
mount /dev/sdc1 /media

# eject a device using the umount command (umount not unmount)

umount /dev/sdb1
umount /dev/sdc1
```

### Getting Information on Mounted Disks

The command _df_ (disk free) 2ill provide us with basic information on any hard disks or mounted devices

```
# list mounted devices

df

# add the '-h' switch for human-readable sizes

df -h
```

## The Logging System

Linux uses a deamon called _syslogd_ to automatically log events on your computer.

In the _rsyslog.cong_ file, there is a _RULES_ section. This sections determines what kind of information is logged, what programs have their message logged, and where that log is stored.

### Automatically Cleaning Up Logs with logrotate

Since log files take up space, _log rotation_ is the process the help us archiving log files by moving them to some other location, leaving you witha fresh log file. This archived location will then get cleaned up after a specific period of time.

Your system is already rotating log files, you can find the configuration the configuration file in _/etc/logrotate.conf_.

### Removing Evidence

Deleted files are still available until they are overwritten. Linux has a built-in command to delete a file and overwrite it severl times, making it much harder to recover, the command is _shred_.

By default _shred_ overwrite a file 4 times. Generally, the more times the file is overwritten, the harder it is to recover, but keep in mind that each overwrite takes time.

```

# -n how many times to overwrite the file

shred -f -n 10 /var/log/auth.log.*
```

Once you have shred a file, you will se that the contentents are indecipherable gibberish.

Another way to for covering your tracks is to simply disable logging.

## Becoming Secure and Anonymous

### How the Internet Gives Us Away

- Your IP address identifies you as you traverse the internet. Data sent from your machine is generally tagged with your IP address, making your activities easy to track

- Google and other email services will "read" your email, looking for keywords to more efficiently serve you ads

- When you send a packet of data across the internet, it contains the IP addresses for the source and destination for the data. Anyone intercepting the packet can see who sent it, where it has been, and where it's going

Each packet hops through multiple internet routers until it finds its destination and then hops back to the sender.

To see what hops a packet might make between you and the destination, you can use the _traceroute_ command:

`traceroute google.com`

### The Onion Router System (Tor)

In the 1990s, the US Office of Naval Research (ONR) set out to develop a method for anonymously navigating the internet for espionage purposes. The plan was to set up a network of routers that was separate from the internet's routers, that could encrypt the traffic, and that only stored unencrypted IP address of the _previous_ router -- meaning all other router addresses along the way were encrypted. The idea was that anyone watching the traffic could not determine the origin of destination of the data.

Packets sent over Tor are not sent over the regular routers so closely monitored by so many but rather are sent over a network of over 7000 routers around the world, thanks to volunteers who allow their computers to be used by Tor.

On top of using totally separate router network, Tor _encrypts the data, source, and destination IP address of each packet_.

> The Tor browser is capable of accessing the _dark web_. The websites that make up the dark web require anonymity, so they allow access only through the Tor browser, and they have addresses ending in _.onion_ for their top-level domain (TLD).

### Security Concerns

The intelligence and spy services of the United States and other nations consider the Tor network a threat to national security, believing such an anonymous network enables foreign governments and terrorists to communicate without being watched.

> Tor's anonymity has been broken before by these authorities and will likely be broken again. The NSA, as one instance, runs its own Tor routers, meaning that your traffic may be traversing the NSA-s routers when you use Tor. The NSA also has a method known as _traffic correlations_, which involves looking for patterns in incoming and outgoing traffic that has been able to break Tor's anonymity.

### Proxy Servers

Another strategy for achieving anonymity on the internet is to use _proxies_, which are intermediate systems that act as middlemen for traffic: the user connects to proxy, and the traffic is given the IP address of the proxy before it's passed on.

> Of course, the proxy will likely log your traffic, but an investigator would have to get a subpoena or search warrant to obtain the logs.

To make your traffic even harder to trace, you can use more than one proxy, in a strategy known as a _proxy chain_.

Kali Linux has an excellent proxying tool called _proxychains_ that your can set up to obscure your traffic.

```
# proxychains <command> <argument>

proxychains nmap -sT -Pn <IP address>
```

You can set up a proxy list by configuring the `[ProxyList]` section in the `/etc/proxychains.conf` file.

### Dynamic Chaining

_Dynamic chaining_ runs our traffic through every proxy on our list and, if one of the proxies is down or not responding, automatically goes to the next proxy in the list without throwing an error. You can enable this configuration in the _proxychains.conf_ file.

### Random Chaining

With _random chaining_ option, proxychains will randomly choose a set of IP addresses from our list and use them to create our proxy chain. This means that each time we use proxychains, the proxy will look different to the target, making it harder to track our traffic from its source. You can enable this configuration in the _proxychains.conf_ file.

> Proxychains is only as good as the proxies you use. If you are intent on remeaining anonymous, do _not_ use a free proxy.
> "If something is free, you are not the customre; you are the product." Bruce Schneier.

### Virtual Private Networks

Using a _virtual private network (VPN)_ can be an effective way to keep your web traffic relatively anonymous and secure. A VPN is used to connect to an intermediary internet device such as a router that sends yout traffic to its ultimate destination tagged with the IP address of the router.

> Using a VPN it's not a guarantee of anonymity. The internet device you connect to must record or log your IP address to be able to properly send the data back to you, so anyone able to access these records can uncover information about you.

The traffic between you and the VPN device is encrypted, so even your internet service provider can't see your traffic.

> Use only VPNs that promise not to store or log any information (and hope they are being truthful).

### Encrypted Email

Free commercial email services such as Gmail, Yahoo!, and Hotmail are free for a reason: they are vehicles for tracking your interests and serving up advertisements. Google, for example, have access to the unencrypted contents of your email, even if you are using HTTPS.

One way to prevent this is to use encrypted email such as _ProtonMain_ that offer end to end ecnryption so that even the ProtonMail admins can't read your email.

### Summary

- **HTTPS**: Encryption at layer 7 (Web Browser). Source and destination IP address are not encrypted but only the data

- **Proxy**: A way to mask your IP address when navigating to the internet. Use multiple proxies to make it harder to get tracked. No encryption provided, just a hidden IP address.

- **VPN**: Encryption ad layer 3, encrypting all kind of data through the internet not only HTTP traffic. IP address hidden by VPN IP address and end-to-end encryption provided from client to VPN, from VPN to destination, and back.

* **Tor**: Layer encryption for each node/onion router. Each node generates an asymmetric key pair, the original data is encrypted with each private key and each node removes its layer of encryption (like an onion) using its public key. By removing the layer, the source IP address and the next node IP address is revealed. This ensure that nobody will know the entire path of the packet. The traffic is encrypted from client to the exit node; from the exit node to the destination server the traffic is not encrypted by Tor.

### Wi-Fi Recon with Aircrack-ng

Before you can even consider attacking a Wi-Fi AP, you need the MAC address of the target AP (BSSID), the MAC address of a client and the channel the AP is operating on.

> In Wi-Fi communication, data is transmitted using radio waves, and these radio waves are characterized by their frequencies. Wi-Fi networks work on two different frequency bands: 2.4 GHz and 5 GHz. In a 2.4 GHz wireless network there are 14 channels available but only 1, 6 and 11 are non-overlapping, because they have sufficient separation from each other to minimize interference.

> A Wi-Fi channel is a communication line with a specific width. A channel is essenctially a specific range of frequencies within a given frequency band. For example, a common width for a channel in a 2.4 GHz frequency band is 20MHz. It means that a channel may have a range of 2410MHz to 2430MHz. The wider the channel the bigger the amount of data that can be transmitted, but they may also increase the change of interference.

> 5 Ghz frequency band is generally less crowded, offering more available channels and reduced interference, it provides higher data transfer rates. 2.4 GHz frequency band is better at penetrating obstacles like walls, and has slightly better range compared to 5 GHz.

> Bandwidth describes the potential maximum speed of data transfer on a network, while Throughput reflects the real-world, effective speed of data transfer, which may be less than the maximum bandwidth due to various factors like network congestion, signal interference, or protocol overhead.

To put your wireless network card in monitor mode, use the _airmon-ng_ command from the aircrack-ng suite.

```
# airmon-ng start|stop|reset interface`

Bluetooth is a universal protocol for low-power, near-field communication operating at 2.4-2.485 GHz.

The bluetooth specification has a minimum range of 10 meters, but there is no limit to the upper range manufacturers may implement in their devices.

Connecting to bluetooth devices is referred as _pairing_. When two devices pair, they exchange a secret key.

## Managing the Linux Kernel and Loadable Kernel Modules

The _kernel_ is at the center of the operating system and controls everything the operating system does, including managing memory, controlling the CPU, and even controlling what the user sees on the screen.

The second element of the operating system is often referred to as _user land_ and includes nearly everything else.

### What is a Kernel Module?

Linux is a monolithic kernel that enables the addition of kernel modules. As such, modules can be added and removed from the kernel. The kernel will occasionally need updating, which might entail installing new device drivers (such as video cards, Bluetooth devices, or USB devices), filesystem drivers, and even system extensions. These drivers must be embedded in the kernel to be fully functinal.

In some systems, to add a driver, you have to rebuild, compile and reboot the entire kernel, but Linux has the capability of adding some modules to the kernel without going through the entire process. These modules are referred to as _loadable kernel modules_, or LKMs.

So, if a hacker can successfully tempt a Linux admin into installing a video or other device driver that has a rootkit embedded in it, the hacker can take total control of the system and kernel.

> A rootkit is a type of malicious software (malware) designed to gain unauthorized access to a computer or network and obtain privileged access, often at the root or administrative level. Once installed on a system, a rootkit can hide its presence and activities, making it difficult for regular security measures to detect or remove it. Rootkits on Linux systems often utilize kernel modules to achieve their goals.

### Checking the Kernel Version

```
# it is possibile to check the kernel version using this two commands:
uname -a

cat /proc/version
```

### Kernel Tuning with sysctl

With the right commands, you can _tune_ your kernel, meaning you can change memory allocations, enable networking features. and even harden the kernel against outside attacks.

Modern Linux kernels use the `sysctl` command to tune kernel options. To make any changes permanent, you have to edit the config file for `sysctl` directly at _/etc/sysctl.conf_.

### Finding More Info with modinfo

```
# to learn more about any kernel modules, use the modinfo command

modinfo bluetooth
```

### Adding and Removing Modules with modprobe

```
# add a module to your kernel

modprobe -a <module name>

# remove a module using -r

modprobe -r <module name>
```

> There is another old command for managing kernel modules in Linux, the insmod command. It's important to know that modprobe is safer than insmod. modprobe understands dependencies, options, and installation and removal procedures and it takes all of these into account before making changes.

## Automatic Tasks with Job Scheduling

Like anyone using Linux, the hacker ofter has jobs, scripts or other tasks, that they want to run periodically.

### Scheduing an Event or Job

The cron daemon (`crond`) and the cron table (`crontab`) are the most useful tools for scheduling regular tasks. The cron daemon checks the cron table for which commands to run at specified times.

To schedule these tasks or jobs, enter them into the cron table file, located at _/etc/crontab_. The cron table has seven fields: the first five are used to schedule the time to run the task (minute, hour, day of the month, moth, day of the week), the sixth field specifies the user, and the seventh field is used for the absolute path to the command you want to execute.

```
# scheduling a backup task at /etc/crontab

00 2 * * 0 backup /bin/systembackup.sh
```

### Using rc to Run Jobs at Startup

You can add services for the _rc.d_ script to run at startup using the `update-rc.d` command.

```
# the update-rc.d command

update-rc.d <name of the script or service> <remove|defaults|disable|enable>

# update our rc.d to have PostgreSQL run automatically at bootup

update-rc.d postgresql default
```
